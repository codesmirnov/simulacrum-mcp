# Security Policy

## Supported Versions

We take security seriously and actively maintain security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Simulacrum MCP, please help us by reporting it responsibly.

### How to Report

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Email security reports to: **hello@codesmirnov.ru**
3. Include detailed information about the vulnerability:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We'll acknowledge receipt of your report within 48 hours
- **Investigation**: We'll investigate and work on a fix
- **Communication**: We'll keep you updated on our progress
- **Disclosure**: We'll coordinate disclosure once a fix is ready
- **Credit**: We'll credit you in the security advisory (if desired)

### Security Updates

Security updates will be:
- Released as soon as possible
- Announced via GitHub Security Advisories
- Documented in the CHANGELOG.md
- Tagged with appropriate security version numbers

## Security Best Practices

When using Simulacrum MCP:

1. **Keep dependencies updated**: Regularly update to the latest versions
2. **Use virtual environments**: Isolate your installations
3. **Review permissions**: Only grant necessary permissions to the MCP server
4. **Monitor logs**: Keep an eye on simulation outputs for anomalies
5. **Validate inputs**: Always validate simulation parameters and data
